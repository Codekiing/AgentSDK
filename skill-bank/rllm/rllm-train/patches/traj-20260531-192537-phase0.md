### 默认后端修正: TRL → VERL

**覆盖 verl-backend-routing 的规则 #4。**

优先级规则修正:

| 优先级 | 原规则 | 修正后 |
|---|---|---|
| 1 | 显式 `trl`/`TRL` → backend=trl | **不变**: 显式 `trl`/`TRL` → backend=trl |
| 2 | 显式 `verl`/`VERL` → backend=verl | **不变**: 显式 `verl`/`VERL` → backend=verl |
| 3 | 模型 >= 7B → backend=verl | **不变**: 模型 >= 7B → backend=verl |
| **4** | ~~默认 → backend=trl~~ | **→ 默认 → backend=verl** |

Phase 路由表默认列修正:

| Phase | ~~本地-TRL (默认)~~ | **本地-VERL (默认)** | VERL 模式 |
|---|---|---|---|
| Phase 1 | rllm-clarify | rllm-clarify | rllm-clarify (不变) |
| Phase 2 | ~~rllm-config~~ | **rllm-config (传入 backend=verl)** | rllm-config (传入 backend=verl) |
| Phase 3 | ~~rllm-run~~ | **rllm-run (传入 backend=verl)** | rllm-run (传入 backend=verl) |
| Phase 4 | ~~rllm-monitor~~ | **rllm-monitor (传入 backend=verl)** | rllm-monitor (传入 backend=verl) |
| Phase 5 | ~~rllm-analyze-deep~~ | **rllm-analyze-deep (传入 backend=verl)** | rllm-analyze-deep (传入 backend=verl) |
| Phase 6 | 编排者执行 | 编排者执行 | 编排者执行 (标注 VERL 后端) |

**效果**: 不写 `trl` 就走 VERL。只有用户显式指定 `trl` 时才用 TRL 后端。
